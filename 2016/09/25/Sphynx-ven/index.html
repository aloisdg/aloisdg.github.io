Translating a book. (Part I Tooling)

  2016-09-25

I am going to start the translation of a french tabletop game called Sphynx. I want to try to use only Open Source software. Lets see what I need.

The Mission

The 1100+ words books a docx file. I guess written with Microsoft Word. The author also share with us a PDF export of the book. We are three persons working together: two translators and the original author. So, we are going to convert the docx to text and convert it together.

The stack

  • Pandoc [GPL] for converting
  • Visual Studio Code [MIT] for writing
  • Git [GPL] for versionning
  • GitLab [MIT] for hosting
  • GitBook [Apache] for the architecture

Roadmap

  • [ ] Create project on GitLab
  • [ ] Host GitBook on GitLab
  • [ ] Convert from .docx to .md
  • [ ] Format the french version to feat GitBook architecture
  • [ ] Push the french version
  • [ ] Start Translating on a different branch

Bonus

I may use Wekan, if need another level of organisation and a task manager. I think GitLab’s issues will be enough. We will see.
